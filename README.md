# HealthNG AI — Nigeria Health Assistant

A production-ready Flask application powered by **Gemini 1.5 Flash** (free API tier).
The API key lives **only on the server** in a `.env` file — users never see or touch it.

## Local Development

### 1. Clone & enter the project
```bash
git clone https://github.com/YOUR_USERNAME/healthng-ai.git
cd healthng-ai
```

### 2. Create a virtual environment
```bash
python -m venv venv
source venv/bin/activate        # Mac / Linux
venv\Scripts\activate           # Windows
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Set up your .env file
```bash
cp .env.example .env
```

Edit `.env` and fill in your values:
```
SECRET_KEY=paste-a-long-random-string-here
GEMINI_API_KEY=paste-your-gemini-api-key-here
FLASK_ENV=development
PORT=5000
```

Get a **free** Gemini API key at: https://aistudio.google.com/app/apikey
Generate a secret key with:
```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

### 5. Run the app
```bash
python app.py
```

Visit: http://localhost:5000

## Deploy to Render (Free)

### Step 1 — Push to GitHub
```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/healthng-ai.git
git push -u origin main
```

> ✅ `.gitignore` already excludes `.env` — your secrets are safe.

### Step 2 — Create a Render account
Go to https://render.com → Sign up (free).

### Step 3 — New Web Service
1. Click **New → Web Service**
2. Connect your GitHub repository
3. Render reads `render.yaml` automatically — build & start commands are pre-configured

### Step 4 — Set environment variables in Render dashboard
In your Render service → **Environment** tab, add:

| Key | Value |
|-----|-------|
| `SECRET_KEY` | Auto-generated by Render (already in render.yaml) |
| `GEMINI_API_KEY` | Paste your free Gemini API key here |
| `FLASK_ENV` | `production` |

> **Never** put your API key in `render.yaml` or any file tracked by Git.

### Step 5 — Deploy
Click **Save Changes** → Render redeploys automatically.
Your app will be live at: `https://healthng-ai.onrender.com`

## API Reference

All endpoints return JSON.

| Method | Endpoint | Body | Description |
|--------|----------|------|-------------|
| GET | `/api/status` | — | Check AI readiness |
| POST | `/api/symptom-check` | `{symptoms, age, gender, state, duration, notes}` | Analyze symptoms |
| POST | `/api/chat` | `{message, history}` | Multi-turn consultation |
| POST | `/api/disease-info` | `{disease}` | Get disease details |
| POST | `/api/prevention` | `{topic}` | Get prevention guide |
| POST | `/api/upload-doc` | `multipart/form-data` (file) | Upload PDF / TXT |
| POST | `/api/query-docs` | `{query, general}` | Query uploaded documents |
